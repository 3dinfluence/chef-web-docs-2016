application_nginx Lightweight Resource¶

Warning

DRAFT

The application_nginx cookbook is used to manage Nginx applications.

The application_nginx cookbook contains the following lightweight resource: application_nginx_load_balancer.

application_nginx_load_balancer¶

The application_nginx_load_balancer lightweight resource is used to install and configure Nginx with an upstream that points to all the nodes returned by a search for a specific role.

Note

This lightweight resource is part of the application_nginx cookbook (http://community.opscode.com/cookbooks/application_nginx).

Actions¶

This lightweight resource provider has the following actions:

Action Description
:deploy Use to deploy an application.
:force_deploy Use to remove any existing release of the same code version and re-deploy a new one in its place.

Attributes¶

This lightweight resource provider has the following attributes:

Attribute Description
application_port The port on which the application server will run. Default value: 8080.
application_server_role The name of a role for an application server. During a Chef run, a search will be performed to find application servers. Default value: #{application name}_application_server.
port The port to which Nginx will bind. Default value: 80.
server_name The virtual host name for a server. Defaults to the node The FQDN of the host..
ssl Indicates whether SSL is being used. Default value: false.
ssl_certificate The SSL certificate used to verify communication over HTTPS.
ssl_certificate_key The certificate key used for SSL communication.
static_files A hash that maps URLs to files. Default value: {}.
template The name of a template that will be rendered to create a context file. Default value: load_balancer.conf.erb (from this cookbook).

Examples¶

None.
