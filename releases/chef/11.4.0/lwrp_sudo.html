sudo Lightweight Resource¶

The sudo cookbook is used to install Indicates that a bootstrap operation should be executed using sudo. and configure /etc/sudoers.

The sudo cookbook contains the following lightweight resource: sudo.

sudo¶

The sudo lightweight resource is used to manage sudo fragment files in /etc/sudoers.d. This lightweight provider has two modes:

  • Natural mode mimics the sudo file interface. This mode will render a sudo fragment in /etc/sudoers.d
  • Template mode requires an erb template and a hash of variables. This mode will render a sudo fragment in /etc/sudoers.d and ensure that any resulting sudo fragments pass validation and have the proper permissions to the file system.

In the event that a sudo fragment does not pass validation, this lightweight resource will fail the Chef run before the fragment can be copied to /etc/sudoers.d. This will prevent corruption of your sudo configuration.

Note

sudo version 1.7.2 (or newer) is required to use the sudo lightweight resource, as it relies on the #includedir directive that is introduced with version 1.7.2. The recipe does not enforce installing the version. To use this lightweight resource, set node['authorization']['sudo']['include_sudoers_d'] to true.

Note

This lightweight resource is part of the sudo cookbook (http://community.opscode.com/cookbooks/sudo).

Actions¶

This lightweight resource provider does not have any actions.

Attributes¶

This lightweight resource provider has the following attributes:

Attribute Description
commands An array of commands that the user or group can execute using sudo. This value must use the full path for each command, otherwise the resulting fragment will fail validation.
group The group for which sudo privileges are provided.
host  
nopasswd Indicates whether a password must be supplied when invoking sudo.
runas  
template A erb template file in the current cookbook (not the sudo cookbook). If this attribute is used, all other attributes will be ignored except for variables.
user The user for which sudo privileges are provided.
variables A hash of variables that are passed into an erb template file.

Examples¶

To use the natural (default) mode, do something like:

sudo "tomcat" do
  user "%tomcat" # or a username
  runas "app_user" # or "app_user : tomcat"
  commands ["/etc/init.d/tomcat restart"] # array of commands, will be .join(",")
  host "ALL"
  nopasswd false # true prepends the runas_spec with NOPASSWD
end

which will generate something like the following in /etc/sudoers.d/tomcat:

# this file was generated by chef
%tomcat ALL=(app_user) /etc/init.d/tomcat restart

To use the template mode, do something like:

sudo "tomcat"
  # this template must exist in the calling cookbook
  template "restart_tomcat.erb"
  variables( :cmds => [ "/etc/init.d/tomcat restart" ] )
end

which will generate something like the following in /etc/sudoers.d/tomcat:

# this file was generated by chef
%tomcat ALL=(app_user) /etc/init.d/tomcat restart
