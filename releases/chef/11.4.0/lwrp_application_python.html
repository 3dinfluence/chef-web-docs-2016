application_python Lightweight Resource¶

Warning

DRAFT

The application_python cookbook is used to manage Celery, Django, and Gunicorn applications.

The application_python cookbook contains the following lightweight resources: application_python_celery, application_python_django, and application_python_gunicorn.

Note

These lightweight resources are part of the application_python cookbook (http://community.opscode.com/cookbooks/application_python).

application_python_celery¶

The application_python_celery lightweight resource is used to deploy Celery task queues in a Python environment.

Actions¶

This lightweight resource provider has the following actions:

Action Description
:deploy Use to deploy an application.
:force_deploy Use to remove any existing release of the same code version and re-deploy a new one in its place.

Attributes¶

This lightweight resource provider has the following attributes:

Attribute Description
config  
template  
django  
celeryd  
celerybeat  
celerycam  
camera_class  

Examples¶

None.

application_python_django¶

The application_python_celery lightweight resource is used to deploy Django in a Python environment.

Actions¶

This lightweight resource provider has the following actions:

Action Description
:deploy Use to deploy an application.
:force_deploy Use to remove any existing release of the same code version and re-deploy a new one in its place.

Attributes¶

This lightweight resource provider has the following attributes:

Attribute Description
collectstatic Indicates the behavior of the staticfiles application. If true, it will invoke manage.py with collectstatic -- noinput. Default value: false.
database_master_role The name of a role. During a Chef run, a search will be performed to find a node with the same name as this value in the same environment as the current role. If a node is found, it’s IP address will be used when rendering the context file.
debug Indicates that the default settings template will control debugging. Default value: false.
legacy_database_settings Indicates that a default settings template will be generate legacy database configuration variables. Default value: false.
local_settings_file The name of the local settings file to be generated by a template. Default value: local_settings.py.
packages An array of pip package to install.
requirements The relative path to a requirements file. If this attribute is unspecified, the provider will look for one in the project root named requirements/#{chef_environment}.txt or requirements.txt.
settings A hash of additional settings that will be made available to a template.
settings_template The name of a template that will be rendered to create the local settings file. If specified, this file must be located in the application cookbook. Default value: settings.py.erb (from this cookbook).

Examples¶

None.

application_python_gunicorn¶

The application_python_celery lightweight resource is used to deploy the Gunicorn worker model in a Python environment. For more information, see: http://gunicorn.org/configure.html.

Actions¶

This lightweight resource provider has the following actions:

Action Description
:deploy Use to deploy an application.
:force_deploy Use to remove any existing release of the same code version and re-deploy a new one in its place.

Attributes¶

This lightweight resource provider has the following attributes:

Attribute Description
app_module Set this value to django to run a Django application; any other string or symbol will build a Gunicorn base command.
backlog The maximum number of pending connections.
daemon Indicates that the Gunicorn process is daemonized.
debug Indicates whether debugging is turned on. Default value: false.
host  
keepalive The amount of time (in seconds) to wait for requests on a Keepalived connection.
logfile A file to which log entries are written.
loglevel The granularity of error logging: debug, info, warning, error, or critical.
max_requests The maximum number of requests a worker will process before restarting.
pidfile A file name to use for the PID file.
port  
preload_app Indicates that application code is loaded before worker processes are forked. Default value: false.
proc_name The name of an instance of Gunicorn. This attribute is only necessary when more than one instance of Gunicorn is running.
settings_template A template to render to create the gunicorn_config.py file. Default value: se.py.erb (from the gunicorn cookbook).
timeout The amount of time (in seconds) that a worker can be silent before it is killed and restarted.
trace Indicates whether trace functions are executed by the server. Default value: false.
umask A bit mask used for the file mode on files written by Gunicorn.
worker_class The type of worker to use: sync, eventlet, gevent, or tornado.
worker_connections The maximum number of simultaneous clients.
workers The number of allowed worker processes.

Examples¶

None.
